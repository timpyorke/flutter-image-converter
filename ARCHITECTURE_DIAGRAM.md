## Component Responsibilities

### Views ğŸ¨

- **Responsibility**: Present data and capture user input
- **Should**: Use Consumer/Provider to access ViewModels
- **Should NOT**: Contain business logic or state
- **Example**: Display buttons, forms, images, text

### ViewModels ğŸ§ 

- **Responsibility**: Manage state and coordinate business logic
- **Should**: Extend ChangeNotifier, call services, notify listeners
- **Should NOT**: Import Flutter widgets or access context directly
- **Example**: Handle button press logic, manage loading states

### Services âš™ï¸

- **Responsibility**: Execute business operations
- **Should**: Be reusable, return results, handle errors
- **Should NOT**: Know about UI or state management
- **Example**: Image processing, API calls, file operations

### Models ğŸ“¦

- **Responsibility**: Define data structures
- **Should**: Be immutable when possible, have copyWith methods
- **Should NOT**: Contain business logic
- **Example**: ImageData class with properties
